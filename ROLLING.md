- 灰度模块是 **亚博八路巡线模块**，用户当前只接 6 路：`X1/X2/X4/X5/X7/X8`，`X3/X6` 不接。

- 用户现场描述：检测到黑线时模块灯亮，OLED `G` 对应位变 `0`，`N+1`；模块有效检测距离约 3~4mm。
- 当前代码状态仍需复核：`Grayscale_Read()` 现在是低电平置 bit=1，`N` 和 `E` 按 bit=1 处理；这与用户描述“G 位变 0 但 N+1”可能语义冲突。
- 模块要求 5V，用户用外部 5V 供电；必须共地。若输出高电平为 5V，MSPM0 GPIO 有风险。用户当前选择先继续调代码。

## 今天 / 下一轮最该做什么

1. 先不要接入第一题，继续灰度静态诊断。
2. 固定模块离地约 3~4mm，在白底、单路黑线、多路/横线下记录 OLED：`G/E/N`。
3. 重点确认：
   ```text
   黑线灯亮时 G 对应位到底显示 0 还是 1
   N 是否真的随着黑线路数增加
   E 左侧黑线是否为负，右侧黑线是否为正
   白底贴近赛道时是否能稳定 N=0
   ```
4. 如果 `G` 是 0 表示黑线，而 `N` 要 +1，则代码应统一成“显示原始电平，统计 0 的数量”；不要再把显示语义和控制语义混在一起。
5. 若继续验证 IMU，不要上车跑 PID，先做静态轴映射诊断：显示 `CMD=0x06` R/P/Y、`CMD=0x02` D0/D1/D2，按正中、右转30°、回正、左转30°、回正、抬头、压头、侧倾测试。
6. 复核官方姿态角时按说明书比例 `raw / 32768 * 180`，当前代码 `raw * 0.01f` 只作为旧诊断结果参考。
7. 灰度通过后，再把 Task 1 从诊断页切回直线控制：编码器 + D0 跑 A→B，接近终点后灰度辅助停车。
