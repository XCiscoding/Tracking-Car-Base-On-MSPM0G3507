# 小车直线实验记录

> 用于记录自动小车直线行驶测试数据、现象和后续改进依据。每次改参数后新增一组记录，不覆盖旧数据。

## 记录字段说明

- 目标距离：程序设定的停车距离，单位 mm。
- 实测距离：地面尺量的小车实际行驶距离，单位 mm。
- OLED D：程序根据编码器估算的距离，单位 mm。
- L/R 脉冲：停车后 OLED 显示的左右编码器脉冲。
- 偏转方向：以小车前进方向为准，记录左偏或右偏。
- 终点偏角：停车后车身相对起点方向的角度估计。
- 异常现象：蛇形、突然大角度偏转、打滑、单轮明显快等。
- 参数：本轮测试的 BASE、TRIM、KP、目标距离等。

## 2026-05-22｜直线测试 V1.1

### 程序参数

```c
#define STRAIGHT_TEST_MM        260u
```

### 现场现象

- 小车非常不稳定。
- 大多数情况往右偏。
- 最终偏转角度大约 15°。
- 偶尔偏转角度大于 60°。
- 地面实测行驶距离约 600mm–700mm，明显大于 OLED D 的 274mm–279mm。

### 初步结论

1. OLED D 稳定在 274mm–279mm，说明程序停车条件能触发，编码器读数在软件内部比较稳定。
2. 地面实测 600mm–700mm，说明当前编码器距离换算严重低估实际距离，误差约 2.2–2.5 倍。
3. 多数测试 L 脉冲大于 R 脉冲，说明左轮记录到的运动量更大；结合右偏现象，左轮实际推进可能更强。
4. 偶发 60° 大偏转不是单纯比例参数问题，更像是左右轮有效驱动力、编码器计数、地面摩擦或 IMU 纠偏方向/延迟存在不稳定因素。

### 下一轮建议记录

- 增加左右轮独立距离或脉冲显示。
- 降低速度到 BASE=200–220。
- 暂时关闭 IMU 纠偏，只做固定左右 PWM 校准，先让机械开环尽量直。
- 用 300mm 实测距离反推有效每毫米脉冲数，再更新距离换算。

## 2026-05-22｜开环标定 V1.2

### 本轮目的

关闭 IMU 纠偏，只用固定左右 PWM 前进。先判断底盘自然偏差和编码器距离比例，不让闭环控制掩盖问题。

| 1 | 287 | 520 | 511 | 600 | 往右偏 15° | -5 | 左右脉冲接近，实际距离约为 OLED D 的 2.09 倍 |
| 2 | 289 | 524 | 513 | 700 | 往右偏 10° | 0 | 实际距离约为 OLED D 的 2.42 倍 |
| 3 | 288 | 519 | 514 | 730 | 先左偏 5°，再右偏 15°，最终右偏 10° | 0 | 出现方向摆动，实际距离约为 OLED D 的 2.53 倍 |

| 5 | 289 | 528 | 511 | 700 | 最终往右偏 30° | 0 | 右偏明显，L-R=+17 |

### 判断规则

- 若仍明显右偏：左轮仍偏快或右轮偏慢，继续降低左轮 PWM，或提高右轮 PWM。
- 若变成左偏：`LEFT_TRIM=-55` 过量，回调到 `-45` 左右。
- 若 OLED D 仍约 260mm，但实测仍 600mm 以上：优先重标定 `ENCODER_PPR`，不要继续调直线闭环。
- 若左右脉冲接近但车仍大角度偏：检查轮胎打滑、万向轮、重心、地面摩擦或电机安装。

### 本轮结论

1. OLED D 稳定在 287mm–289mm，说明程序停车点稳定。
2. 实测距离 600mm–730mm，约为 OLED D 的 2.09–2.53 倍，编码器距离换算仍严重低估。
3. 左右脉冲非常接近，多数为 L 比 R 多 5–17 个脉冲，说明当前 `L185/R240` 已经把左右编码器计数拉近。
4. 车仍大多右偏，且第 3 次出现先左后右，说明问题不只是左右 PWM 静态比例，可能还包含启动瞬间差异、轮胎打滑、地面摩擦、车身重心或万向轮扰动。
5. 终点 Yaw×10 基本为 0，但肉眼偏角明显，说明 IMU Yaw 显示/安装轴向/更新方式不能直接当作当前偏角真相源，需要单独验证。

### 下一步决策

- 先把距离换算修准：按实测距离和平均脉冲反推，有效 `DIST_PER_PULSE` 约为 1.17–1.42mm/pulse。
- 取中间保守值 `DIST_PER_PULSE≈1.30mm/pulse`，对应 `ENCODER_PPR≈116`（150.8 / 1.30）。
- 下一版先把 `ENCODER_PPR` 从 270 改为 116，并把目标距离设为 300mm，再做开环复测。

## 2026-05-22｜编码器 P 闭环 V1.4/V1.5：第一层闭环验证

### 控制目标

第一层只负责让左右轮累计脉冲尽量一致：

```text
error = left_pulse - right_pulse
corr  = error * ENC_KP
left_pwm  = base_left  - corr
right_pwm = base_right + corr
```

该层不能直接知道车头角度，只能修正左右轮走得不一样导致的偏差。

### 有效参数

```c
#define CAL_TARGET_MM           500u
#define CAL_BASE_PWM            220
#define CAL_LEFT_TRIM           (-50)
#define CAL_RIGHT_TRIM          0
#define ENC_KP                  2
#define ENC_MAX_CORR            80
```

实际基础 PWM：左轮 170，右轮 220。

### 早期强闭环数据

| 1 | 610 | 403 | 392 | +11 | 520 | 右偏10° | -277 |

### 加 E/C 显示后的最终验证

| 次数 | OLED D/mm | L | R | E=L-R | C | 实测距离/mm | 偏转 | Yaw×10 |
|---:|---:|---:|---:|---:|---:|---:|---|---:|
| 1 | 573 | 377 | 370 | 6 | 12 | 510 | 车头几乎无偏移 | 0 |

| 5 | 570 | 371 | 371 | -1 | -2 | 510 | 左偏约30° | 5 |

### 本轮结论

1. 编码器第一层闭环已经有效：最终 `E` 基本压到 -1~7，`C` 在 -2~14。
2. 但 `E≈0` 时车仍会左偏或右偏 15°~30°，说明“左右轮走得一样多”不等于“车头方向正确”。
3. 继续加编码器 Kp 不能解决车头姿态偏差，反而可能放大噪声和扰动。
4. 下一层必须使用 IMU/灰度等车身航向或位置反馈。

## 2026-05-22｜IMU Yaw 诊断

### 当前诊断页

```text
Raw: 原始 Yaw ×10
Zro: 清零基准 ×10
Rel: 相对 Yaw ×10
CMD: 最后 checksum 通过帧命令
OK:  checksum 通过帧数
```

### 测试数据

| 场景 | Raw×10 | Zro×10 | Rel×10 | CMD/OK 现象 | 判断 |
|---|---:|---:|---:|---|---|
| 静止 | - | - | 3/4 跳 | - | 静止抖动约 0.3°~0.4°，可接受 |
| 右转约30° | -6 | 0 | -6~-8 | CMD 0/85 切换，OK 约 58872 | Yaw 只变化约 0.6°~0.8°，不符合真实水平转角 |
| 左转约30° | 10 | 0 | 10 | CMD 195/85 切换，OK 约 68892 | Yaw 只变化约 1°，不符合真实水平转角 |
| 左转约80° | - | - | 约 1700 | - | 当前 Yaw 在某些角度附近跳变到约 170° |

### 本轮结论

1. IMU 串口不是完全不通，OK 计数能增长，`0x55` 四元数帧在流动。
2. 当前 Yaw 静止时稳定性尚可，但它不是车体水平转角的可靠输出。
3. 不能把当前 Yaw 直接接入航向 PID。
4. 下一步要改成 Roll/Pitch/Yaw 三轴相对角诊断，找出真正随水平转车变化的轴。

## 2026-05-23｜IMU 协议与 D0 航向候选确认

### 本轮目的

从 RAW 诊断进入真实帧解析，确认 ATK-IMU901 当前输出协议，并从多个 `CMD` 的原始通道中找出可用于水平航向反馈的候选值。

### 协议结论

```text
真实帧格式：55 55 CMD LEN DATA... SUM
CMD 切换：0x01 / 0x02 / 0x03 / 0x06
OK：持续增长，checksum 解析已通
```

已证伪：

```text
AA FF CMD LEN DATA SUM 不是当前模块实际输出
CMD=0x06 的传统 Roll/Pitch/Yaw 不能代表水平车头航向
```

### 候选通道手动测试

OLED 当时显示顺序：

```text
D0 = CMD=0x02[0] - bias
G1 = CMD=0x02[1]
G2 = CMD=0x02[2]
C0 = CMD=0x03[0]
C1 = CMD=0x03[1]
```

| 场景 | D0 | G1 | G2 | C0 | C1 | 判断 |
|---|---:|---:|---:|---:|---:|---|
| 正中静止 | 0 | -346 | 233 | -80 | -190 | D0 回零 |
| 右转约30° | -3012 | -288 | 341 | -90 | -200 | D0 大幅变化 |
| 回正 | -75 | -336 | 235 | -86 | -190 | D0 接近回零 |
| 左转约30° | 645 | -346 | 153 | -80 | -175 | D0 与右转反向 |

### 本轮结论

1. `D0` 是当前唯一明显满足“水平转动有响应、左右方向可区分、回正接近 0”的通道。
2. `G1/G2/C0/C1` 不适合作为当前航向反馈。
3. 后续直线控制以 `IMU_GetHeadingGyroRaw()` 返回的 D0 作为第二层航向误差。

5. 建议把 OLED 刷新从 500ms 改成 200ms，专门观察起步瞬间 `D0/E/EC/HC/LP/RP` 是否跳变。

## 2026-05-23｜IMU 可行性复测方案：姿态角与 D0 轴映射

### 本轮目的

重新确认 ATK-MS901M/IMU901 的官方姿态角、D0/D1/D2 等通道到底哪个对应小车水平航向。该轮只做静态手动测试，不上车跑电机，不接 PID。

### 说明书依据

ATK-MS901M 说明书示例中，姿态角解算比例为：

```text
roll  = raw / 32768 * 180
pitch = raw / 32768 * 180
yaw   = raw / 32768 * 180
```

当前代码中 `CMD=0x06` 暂按 `raw * 0.01f` 显示，比例和说明书不一致。后续若复核官方姿态角，应先改成说明书比例或同时显示 raw 与换算角度。

### 建议 OLED 显示

```text
R/P/Y: CMD=0x06 姿态角
D0/D1/D2: CMD=0x02 三轴原始/零偏值
C/OK: 最后帧命令与 checksum 通过计数
可选：CMD=0x01 三轴值
```

```c
#define STRAIGHT_D0_DIV         20
#define STRAIGHT_D0_MAX_CORR    80
#define STRAIGHT_D0_VALID_LIMIT 3000
#define STRAIGHT_LOOP_MS        20u
#define STRAIGHT_SOFT_START_MS  150u
#define STRAIGHT_SOFT_PWM       170
#define STRAIGHT_MAX_RUN_MS     3000u
```

### 控制逻辑

```text
车动前：锁定 target_d0
0~150ms：base=170，EC 启用，HC=0
150ms 后：base=200，EC 启用，HC=-(D0-target_d0)/20，限幅 ±80
如果 abs(D0-target_d0)>3000：本轮 HC=0
停车：距离达到 500mm 或运行超过 3000ms，任一满足就 Motor_Stop()
```

### 本轮理由

1. 取消左轮 -50 初始补偿，先让编码器 EC 负责左右轮同步；最新一版按用户要求继续加大编码器动态补偿到 `KP=4/MAX=120`。
2. 起步软启动降低第一下甩头概率。
3. 上一轮实测 `HC` 基本一直为 0，小车稳定左偏，说明 D0 修正没有实际参与；当前代码已把 D0 修正调为 `-(D0-target_d0)/20`，最大 ±80，用于验证陀螺仪 P 是否能介入纠偏。
4. D0 曾出现 `15696` 级跳变，因此仍保留 `abs(error)>3000` 时 HC=0 的过滤。
5. OLED 显示改为 `D0/E/EC/HC/M/D/LP/RP`，用于判断 EC/HC 的介入时机和左右 PWM 输出方向。

### 本轮结论

1. 小车从偶发右偏变成基本稳定左偏，说明取消固定补偿后底盘自然偏向暴露出来。
2. `HC` 全部为 0，D0 航向修正没有实际参与；不能据此判断 IMU P 是否有效。
3. `E` 多数很小，编码器动态补偿只是在同步左右脉冲，不能直接保证车头方向。
4. D0 直线调试暂时保留结论，当前主线已切到灰度静态诊断；灰度通过后再恢复 Task 1 直线控制。

## 2026-05-23｜灰度模块移植：亚博八路巡线模块 6 路静态诊断

### 本轮目的

把亚博八路巡线模块先作为静态诊断接入，暂不参与电机控制。目标是先确认接线、G 位顺序、黑白电平、触发数量 N、偏差 E，再决定如何接入 Task 1 的 B 点停车。

### 接线

```text
X1 = PB2
X2 = PB3
X3 = 不接
X4 = PA27
X5 = PB20
X6 = 不接
X7 = PB14
X8 = PB15
```

模块要求 5V，用户使用外部 5V 供电。必须与 MSPM0 共地；若模块输出高电平为 5V，GPIO 有电气风险。

### 当前固件

`empty.c` 当前调用 `Run_Grayscale_Diag()`，不会启动电机。

OLED 显示：

```text
G: X1/X2/X4/X5/X7/X8 六路状态
E: 加权偏差，权重 {-5,-3,-1,+1,+3,+5}
N: 软件统计到的触发路数
```

### 当前代码状态

`Grayscale_Read()` 当前逻辑：读到低电平时将对应 bit 置 1。

```text
X1 -> bit5
X2 -> bit4
X4 -> bit3
X5 -> bit2
X7 -> bit1
X8 -> bit0
```

`Grayscale_GetError()` 当前按 bit=1 的路参与权重计算，`N` 当前也按 bit=1 统计。

### 用户现场反馈

```text
检测到黑线时模块灯亮，G 的对应值变 0，N+1。
模块有效检测距离约 3~4mm。
什么都没探测/离地过远时，也可能灯亮。
```

### 当前结论

1. 亚博模块的有效测试必须贴近赛道 3~4mm，悬空/离地过远时亮灯不能作为赛道有效状态。
2. 用户描述中“G 位变 0 但 N+1”说明显示语义和统计语义可能需要拆开：G 显示原始电平，N 统计低电平触发。
3. 当前代码可能仍与用户描述冲突，下一轮必须先实测白底、单路黑线、多路黑线的 `G/E/N`，再决定是否反相。
4. 未完成前不要接入 Task 1，否则终点检测和左右偏差可能反向。
